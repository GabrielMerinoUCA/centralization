IX. Constructors (Under the Hood)

As we saw on chapter XXIX of the beginner Javascript course, constructor functions are the vanilla way of representing constructors on Javascript. However, there are some mechanics that are not very well explained. First of all, you might remember that we use the 'this' keyword inside the function. Now you might realize that it doesn't make sense as there is no object that the 'this' keyword could be referencing. But in reality, the 'this' keyword in Javascript will always work by referencing the objects that called it. It might sound odd since on definition there is no object calling it but, it is when we instantiate an object that it come to effect which leads us to the second mechanic.

This mechanic would be the 'new' keyword as it has some very interesting properties. You see, the 'new' keyword is capable of transforming functions into objects or constructor functions which return an object again, thanks to the 'new' keyword.

main.js



function User(email, name) {
     this.email = email;
     this.name = name;
     this.login = function() {
          console.log(email + "logged in");
     }
}


let user1 = new User("john@gmail.com", "John Doe");

user1.login();
