VII. Method Chaining

Method Chaining is a very interesting concept that I have already seen on places like android studio with okHttp or retrofit. This concept revolves around being able to call multiple methods in quick succession without the need to recall the object itself every single time.

main.js



class User {
     constructor(email, name) {
          this.email = email;
          this.name = name;
     }

     login() {
          console.log(this.email + " logged in");
          return this; // Returning 'this' allows us to call more methods
     }

     logout() {
          console.log(this.email + " logged out");
          return this;
     }
}


let user1 = new User("john@gmail.com", "John Doe");

user1.login().logout(); // Needless to say that if the function is supposed to return other datatype it won't work

This can be very useful in some scenarios so the question will be, when should we use this?
