XXXVIII. Introduction to Javascript Events

On a web page, almost everything is an event. from all th elements you hover in, the elements you click or drag, etc. A very common event is the "onclick event". This event can be defined as an html attribute or indirectly via Javascript. So which one should you use? well it depends but, for reusability, use the Javascript file approach since if you have to apply an event action for multiple elements, you can do it easier with Javascript

index.html



<body>

    <button> Button 1 </button>

    <button> Button 2 </button>

    <button> Button 3 </button>

</body>


test.js



let parent = document.getElementsByTagName( "body" )[0];

let buttons = parent.getElementByTagName( "button" );


buttons[0].onclick = function() {
     alert("Hello world");
}

buttons[1].onclick = function() {
     alert("Hello world");
}

buttons[2].onclick = function() {
     alert("Hello world");
}

/* You could use a for loop if you wanted to be more practical and actually make this code reusable*/
