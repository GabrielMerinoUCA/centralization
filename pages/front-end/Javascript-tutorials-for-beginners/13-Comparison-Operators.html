XIII. Comparison Operators

13.1. Intermediate expressions

Asides from the operators we already know, there also are the variations such as, the greater-equal than and lesser-equal than operators.

test.js



var myNum = 7;


if(myNum >= 7 ) {
    console.log( "7 is greater or equal to 7" );
}

if(myNum <= 10) {
    console.log( "7 is lesser or equal to 10" );
}


Now going for the more unique Javascript stuff, we'll see the triple equal to operator. As stated previously, Javascript has truthy and falsy values. When we use normal equal to operators, we are only comparing if the result is truthy, so only the values are the same but not the datatype for example. In general, YOU ALWAYS WANT TO USE TRIPLE EQUAL TO OPERATOR.

test.js



var myNum = 5;


if(myNum == "5" ) {
    console.log( "This is true, even if the data types are not the same, they only have to be 'truthy' for '==' to return true." );
}

if(myNum === "5" ) {
    console.log( "Now this won't execute since we are comparing both value and datatype." );
}

if(myNum === 5 ) {
    console.log( "And this statement is true. This is why you should rather use this operator instead." );
}


Of course, Javascript also supports the not equal operator, however it follows the same principle as the triple equal operator and just using "!=" would only check for the value and not the data type.

test.js



var myNum = 4;


if(myNum != "4") {
    console.log( "This should appear on console but since this is falsy, the return value is false." );
}

if(myNum !== "4") {
    console.log( "This will appear since 4 is not a string and we are using the triple not equal to operator." );
}
