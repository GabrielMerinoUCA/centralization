XX. Variable Scope

Variable Scope is an interesting case study on Javascript. By general rule, it works just like in any other programming language. But, there is a little extra twist. If you do not define a variable outside any scope, it will be local. However, it is a little bit more complicated than that... For beginners, we have 2 ways of declaring variables, the let keyword, and the var keyword.

test.js



let z = "z";

function myFunction() {
    {
         var x = "x";
         let y = "y";
    }
     console.log(z); // print "z"
     console.log(x); // print "x"
     console.log(y); // reference error
}


/* To make this even more confusing, you cannot use a "var" declared variable outside a function */

console.log( x );// Reference Error


So what can we conclude from this? Well, do not use the var keyword at all. The scope ambiguity makes it very unpredictable and therefore hard to work with and understand. Just use let for most cases and remember var for those 0.1% scenarios where it makes sense to use it.
