XXIX. Constructor Functions

You might think we are about to talk about Constructor as a concept, but you'll be wrong. What we are about to see here is neglect of the class concept and instead we'll create a function that literally constructs objects for us.

test.js



let Car = function(topSpeed, driver) {
     this.topSpeed = topSpeed;
     this.driver = driver;
     this.drive = function() {
         console.log( "driving" );
    }
}


/* It's called constructor function because it uses the "new" keyword besides "Car" being a function variable.*/

let myCar = new Car(120, "John");

myCar.drive();


What can we conclude out of this? Well, don't use this. Just use classes, as you should, instead of whatever this unholy anti-pattern is. We will cover classes in a different series.
