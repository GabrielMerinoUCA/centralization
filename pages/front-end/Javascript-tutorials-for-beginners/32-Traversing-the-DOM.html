XXXII. Traversing the DOM

32.1. The "document" object

If we want to retrieve any element from within the document, there is a selection of methods from the document object, an object that essentially represents the document as a whole.

index.html



<body>

     <a class="link"> link 1 </a>

     <a class="link"> link 2 </a>

     <a class="link"> link 3 </a>

     <a class="link"> link 4 </a>

</body>


test.js



/* From the document (root), get all elements that have this class Name */

console.log(document.getElementsByClassName( "link" )); // Prints list of links in object format


32.2. Nested element retrieval

If you retrieve an element from the document and that element has many elements, you can retrieve such elements from the previous one.

index.html



<body>

     <div class="content">

         <a>link 1</a>

         <a>link 2</a>

         <a>link 3</a>

         <a>link 4</a>

    </div>

</body>


test.js



let divTags = document.getElementsByClassName("content");


/* divTags is an object list of html elements, such elements have the same properties as the document object since they are all elements/nodes*/

let anchorTags = divTags[0].getElementsByTagName( "a");


32.3. Retrieve a single element

So far we've been retrieving object lists of html elements. However, if we define ids on our elements, we can retrieve them immediately (This is why the id attribute should be unique)

index.html



<body>

     <div id="the-only-content">

        . . .

    </div>

</body>


test.js



console.log( document.getElementById( "the-only-content" ));

// Prints a single html element instead of an object list
